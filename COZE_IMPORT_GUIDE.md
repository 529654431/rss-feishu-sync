# Coze平台导入指南

## 🎯 快速导入步骤（5分钟）

### 步骤1：登录Coze平台

1. **访问Coze平台**：https://www.coze.cn/
2. **登录账号**（使用手机号/邮箱/微信）

### 步骤2：创建工作流

1. **进入工作流管理**
   - 点击左侧菜单的"工作流"
   - 或直接访问：https://www.coze.cn/workflow

2. **创建新工作流**
   - 点击"新建工作流"按钮
   - 选择"从Git仓库导入"

### 步骤3：配置导入信息

填写以下信息：

| 配置项 | 值 |
|--------|-----|
| **仓库地址** | `https://github.com/529654431/rss-feishu-sync.git` |
| **分支** | `main` |
| **工作流名称** | RSS监测与飞书同步 |
| **描述** | 每小时整点监测RSS链接，将新内容自动写入飞书多维表格 |

3. 点击"导入"按钮

### 步骤4：等待导入完成

- 导入过程约1-2分钟
- 系统会自动：
  - 克隆代码
  - 安装依赖
  - 初始化工作流

---

## 🔧 步骤5：配置环境变量

导入完成后，需要配置环境变量：

1. **进入工作流设置**
   - 点击刚创建的工作流
   - 进入"设置"或"配置"页面

2. **添加环境变量**

在"环境变量"部分，添加以下变量：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `SUPABASE_URL` | `https://xxx.supabase.co` | 你的Supabase项目URL |
| `SUPABASE_KEY` | `your_supabase_key` | 你的Supabase API Key |
| `COZE_WORKSPACE_PATH` | `/workspace` | 固定值，无需修改 |

### 如何获取Supabase凭证？

1. **访问Supabase项目**
   - 登录：https://supabase.com/
   - 进入你的项目

2. **获取URL和Key**
   - 点击左侧菜单"Settings" → "API"
   - 复制以下信息：
     - **Project URL**: `https://xxx.supabase.co`
     - **anon public**: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`

3. **填写到环境变量**

---

## ⚙️ 步骤6：配置定时触发器

1. **进入触发器设置**
   - 在工作流页面，找到"触发器"或"Triggers"标签
   - 点击"添加触发器"或"编辑"

2. **确认配置**

检查以下配置是否正确：

```yaml
类型: Cron
Cron表达式: 0 * * * *
时区: Asia/Shanghai
状态: 已启用
输入参数:
  rss_url: https://cn.investing.com/rss/news_285.rss
  app_token: KqLSb9eeqaOYstsryBMcCiJTnkc
  table_id: tbl6xkic4IuaOH6u
```

3. **点击"保存"**

---

## 🧪 步骤7：手动测试

**重要：在激活定时任务前，先手动测试一次！**

1. **进入测试页面**
   - 在工作流页面，点击"运行"或"测试"按钮

2. **输入测试参数**
   - 可以直接使用默认参数
   - 或自定义测试参数：
     ```json
     {
       "rss_url": "https://cn.investing.com/rss/news_285.rss",
       "app_token": "KqLSb9eeqaOYstsryBMcCiJTnkc",
       "table_id": "tbl6xkic4IuaOH6u"
     }
     ```

3. **点击"运行"**

4. **检查结果**
   - 查看执行日志
   - 确认返回：`{"processed_count": 10, ...}`
   - 打开飞书表格，确认有新记录写入

---

## ✅ 步骤8：激活定时任务

1. **确保手动测试成功**
   - processed_count > 0
   - 飞书表格有新记录

2. **启用触发器**
   - 在触发器设置中，将状态改为"已启用"
   - 或点击"启用"按钮

3. **确认配置**
   - Cron表达式：`0 * * * *`
   - 时区：`Asia/Shanghai`
   - 下次执行时间：下一个整点

---

## 📊 步骤9：验证自动运行

1. **等待下一个整点**
   - 例如现在是22:30，等到23:00
   - 或测试时，等几分钟看是否执行

2. **查看执行历史**
   - 在Coze平台，进入工作流详情
   - 查看"执行历史"或"日志"
   - 确认有自动执行的记录

3. **检查飞书表格**
   - 查看是否有新记录自动写入

## 常见问题

### Q1: 导入失败，提示"找不到入口文件"
**A**: 检查 `.coze` 文件中的 `entrypoint` 是否为 `src/main.py`

### Q2: 运行时提示"ModuleNotFoundError"
**A**: 
- 检查环境变量是否配置完整
- 确认依赖包已安装（系统会自动安装）

### Q3: 飞书表格写入失败
**A**:
- 检查 `app_token` 和 `table_id` 是否正确
- 确认飞书表格有写入权限
- 查看执行日志中的错误信息

### Q4: 定时任务不执行
**A**:
- 确认触发器状态为"已启用"
- 检查Cron表达式：`0 * * * *`
- 检查时区：`Asia/Shanghai`

### Q5: 如何修改RSS源？
**A**:
- 在工作流配置中，修改触发器的输入参数
- 或手动运行时，自定义 `rss_url`
